Laboratorio 1: Introducción a las aplicaciones web, HTML5 y ciclo de vida de los sistemas de información

¿Cuál es la diferencia entre Internet y la World Wide Web?
Internet es la red mundial que conecta computadoras y permite enviar datos entre ellas. Y la world wide web es un servicio que funciona dentro de Internet y nos permite ver información en paginas web usando un navegador

¿Cuáles son las partes de un URL?
URL significa Uniform Resource Locator (Localizador Uniforme de Recursos) El protocolo de red es la primera parte y le dice al sistema qué tipo de conexión usar para comunicarse con el servidor, por ejemplo: HTTP, HTTPS, FTP o mailto El subdominio indica el servicio específico donde se encuentra la información, el más común es www (World Wide Web) El dominio es el nombre del servidor al que se quiere acceder. También muestra el tipo de organización (.com, .edu), el país de origen (.ar, .br, .it), lo que se conoce como extensión o TLD (Top Level Domain)

¿Cuál es el propósito de los métodos HTTP: GET, HEAD, POST, PUT, PATCH, DELETE?
GET: se usa para solicitar información de un recurso específico, no permite el envió de datos a excepción si dichos datos se envían como parámetro en la URL que realiza la petición. La respuesta puede incluir contenido en formatos como HTML, JSON, XML, imágenes o JavaScript.
HEAD: es parecido a GET, pero no devuelve el contenido del recurso, solo el código de respuesta HTTP y los encabezados. Se usa cuando no interesa ver la información completa, sino verificar el estado de un enlace o comprobar si un archivo existe.
POST: envía datos al servidor a través del cuerpo de la petición, se usa principalmente para registrar o crear información. Cada petición puede generar un cambio nuevo en el servidor. Se utiliza comúnmente para enviar datos de formularios, archivos o información en formato JSON.
PUT: es idempotente, lo que significa que repetir la misma petición varias veces produce el mismo resultado. Se usa principalmente para actualizar información existente en el servidor, similar a un UPDATE en una base de datos. Los datos que se envían en una petición PUT suelen ir en el cuerpo del mensaje, por ejemplo en formularios o en formato JSON.
PATCH: se usa para modificar solo una parte de un recurso, en lugar de actualizarlo completo y se enfoca en cambios parciales. Para saber si un servidor acepta PATCH, se puede revisar los encabezados como “Allow”, “Access-Control-Allow-Methods” o “Accept-Patch”.
DELETE: se usa para eliminar un recurso específico del servidor y es idempotente.

¿Qué método HTTP se debe utilizar al enviar un formulario HTML, por ejemplo cuando ingresas tu usuario y contraseña en algún sitio? ¿Por qué?
Se debe usar el método POST, porque permite enviar datos sensibles (como usuario y contraseña) en el cuerpo de la petición y no en la URL, lo que ofrece mayor seguridad y evita que esa información quede visible en el historial o en la dirección del navegador.

¿Qué método HTTP se utiliza cuando a través de un navegador web se accede a una página a través de un URL?
Se utiliza es GET, porque el navegador está solicitando información de un recurso (la página web) al servidor sin enviar datos en el cuerpo de la petición, solo pidiendo que le devuelva el contenido de la página, como una página HTML, imágenes o documentos, permitiendo visualizar el sitio web solicitado.
Un servidor web devuelve una respuesta HTTP con código 200. ¿Qué significa esto? ¿Ocurrió algún error?
Significa que la solicitud se realizó correctamente y el servidor devolvió la información solicitada. No ocurrió ningún error, es un código de éxito que indica que todo funcionó como se esperaba.

¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 404? ¿Por qué?
Sí, es responsabilidad del desarrollador corregirlo, porque un error 404 indica que el servidor no puede encontrar una página que debería estar disponible. Esto suele ocurrir por URLs mal escritas, páginas eliminadas o movidas sin redirección, enlaces rotos o problemas de configuración.

¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 500? ¿Por qué?
Sí, es responsabilidad del desarrollador corregir un error 500, porque este código indica un problema interno del servidor que impide que el sitio funcione correctamente. Si no se corrige, el sitio queda inaccesible para los usuarios, afectando la experiencia y la confiabilidad del servicio.

¿Qué significa que un atributo HTML5 esté depreciado o desaprobado (deprecated)? Menciona algunos elementos de HTML 4 que en HTML5 estén desaprobados.
Significa que se considera obsoleto y ya no se recomienda usarlo, porque HTML es un lenguaje que evoluciona constantemente. Aunque algunos navegadores todavía lo soportan, en el futuro podrían dejar de hacerlo, lo que puede causar errores o que la página no se visualice correctamente. Por eso se sustituyen por alternativas más actuales. Algunos de los elementos de HTML 4 que en HTML 5 están desaprobados son applet que se reemplaza por embed u object, center y font que se reemplazan con estilos en CSS, y big que se introduce con CSS.

¿Cuáles son las diferencias principales entre HTML 4 y HTML5?
HTML4 se enfocaba en la estructura básica, mientras que HTML5 ofrece una web más semántica (se refiere a usar etiquetas que describen qué es el contenido, no solo cómo se ve), interactiva y multimedia, facilitando la creación de aplicaciones modernas.

¿Qué componentes de estructura y estilo tiene una tabla?

  • table: es la etiqueta principal que define la tabla
  • tr: define cada fila de la tabla
  • td: representa una celda de datos
  • th: representa una celda de encabezado
  • thead, tbody y tfoot: agrupan filas en encabezado, cuerpo y pie de la tabla
  • caption: añade un título o leyenda a la tabla
  • col y colspan: permiten hacer referencia y ajustar columnas

    • ¿Cuáles son los principales controles de una forma HTML5?
    Los principales controles de una forma HTML5 son los elementos que permiten ingresar y enviar datos del usuario. Los más importantes son:
    Campos de entrada (input): texto, contraseña, correo, fecha, hora, rango, color y archivo
    textarea: para texto largo
    select: listas desplegables
    Casillas de verificación y botones de opción: para seleccionar opciones
    Botones (button): enviar, reiniciar o ejecutar acciones
    Selectores avanzados: fecha, hora, rango y color
    Carga de archivos (input type="file")
    Autocompletar: sugerencias automáticas al escribir
    fieldset y legend: para organizar los controles

    Sobre el ciclo de vida y desarrollo de los sistemas de información:

    ¿Cuál es el ciclo de vida de los sistemas de información?
    Es el conjunto de etapas por las que pasa un sistema desde su planeación hasta su uso y mantenimiento. Este ciclo es el proceso organizado para crear, implementar y mejorar un sistema que recopila, procesa y transmite información.

    ¿Cuál es el ciclo de desarrollo de sistemas de información?
    Las principales fases del ciclo de vida son:
    Planificación: se definen los objetivos del proyecto, su alcance, viabilidad y riesgos
    Análisis: se determinan los requerimientos del sistema y lo que debe hacer
    Diseño: se establece la estructura y la arquitectura del sistema, evaluando alternativas de implementación
    Implementación: se construye el sistema utilizando las herramientas y tecnologías adecuadas
    Pruebas: se detectan y corrigen errores para asegurar su correcto funcionamiento
    Instalación o despliegue: se prepara el entorno de hardware y software donde operará el sistema
    Uso y mantenimiento: se corrigen fallas, se adapta el sistema a nuevas necesidades y se agregan mejoras
